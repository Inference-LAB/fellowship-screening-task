## What I expect to contribute

The parts of the lab's work I can be immediately useful on are LLM applications
and production MLOps, with agentic pipelines as the area I most want to go
deeper in. My final year project, AgroArc, was an AI agricultural advisory
system I built end to end: scikit-learn models for the recommendation layer, a
FastAPI backend serving them, a React frontend, and the Gemini API wired in to
turn model output into advice a farmer could actually act on. Building it meant
dealing with the parts that don't appear in tutorials, like prompt behaviour
degrading on messy real-world inputs, latency once the LLM call sat in the
request path, and containerising the whole thing so it ran somewhere other than
my laptop. At Genie AI I worked on model evaluation, which is where I learned to
distrust a metric that looks good. I've done enough Docker and CI/CD work to
want the deployment path set up in week one rather than discovered in week five.
Practically, I expect to contribute working backend and integration code,
evaluation harnesses for whatever we're shipping, and the infrastructure
scaffolding that keeps a small team from tripping over its own deploys. I also
run developer communities, including GitHub Copilot Dev Days Swabi and
Kubernetes Community Days Islamabad, which is mostly the skill of hitting
deadlines other people depend on. That is what a build sprint is.
